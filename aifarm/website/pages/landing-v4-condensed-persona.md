```

   - 일반 봇: 없음. 깨끗한 계정. 알고리즘이 의심함.
   - 우리 노드: 수개월간의 기록. 알고리즘이 신뢰함.

3. 신뢰 점수(Trust Score)
   - 일반 봇: 0점. 신규 유저 취급.
   - 우리 노드: 80~95점. 우량 유저 취급.

결론:
알고리즘은 "누가 봤는가"를 봅니다.
우리의 노드가 보면, 알고리즘은 "좋은 영상"이라고 판단합니다.

### Function Spec (For Axon)

```
VIP Channel Auto-Watch System

1. YouTube API 연동
   - 채널 ID 등록
   - Webhook: 새 영상 업로드 감지
   
   ### YouTube API 에러 처리
   - Rate Limit (403): 지수 백오프 (1s → 2s → 4s → ..., 최대 60s)
   - Auth 실패 (401): 관리자 알림, API 키 갱신 필요
   - Not Found (404): 채널 삭제됨, 사용자에게 알림
   - 서버 에러 (5xx): 3회 재시도 후 폴백 (수동 체크)

2. Trigger
   - 새 영상 감지 즉시
   - 600대 노드에 시청 명령 전파

3. Priority
   - VIP 채널 영상 = Priority 0
   - 일반 의뢰 영상 = Priority 1
   - 랜덤 시청 = Priority 2

4. Timing
   - 업로드 후 5분 이내 시청 시작
   - 초반 조회수 급등 → 알고리즘 픽업

5. 데이터 지속성 (Data Persistence)
   - 저장소: Supabase PostgreSQL
   - 스키마: vip_channels, vip_watch_logs, node_trust_scores
   - 보존 기간: 로그 90일, 신뢰 점수 영구
   - 정합성: 트랜잭션 기반 업데이트, 낙관적 락

6. 동시성 제어 (Concurrency/Orchestration)
   - 600대 노드 제어: 배치 처리 (50대씩 12배치)
   - Throttling: 초당 최대 10개 명령
   - 큐잉: Redis 기반 작업 큐 (FIFO)
   - 코디네이터: 단일 마스터, 헬스체크 기반 장애 복구
   - 실패 복구: 미완료 작업 자동 재할당

7. 모니터링 및 로깅
   - 감사 로그: 모든 시청 명령 기록 (user_id, timestamp, target)
   - 알림 임계값: 실패율 >5%, 응답시간 >30s
   - 로그 보존: 1년 (규정 준수)

### 컴플라이언스 제약사항

- 사용자 동의 필수 (약관 체크박스)

```

- [ ] CCTV 그리드 각 항목: aria-label="노드 [번호], [상태], [현재 시청 영상]"

- [ ] 애니메이션 상태: aria-describedby로 현재 진행 상황 설명
```
