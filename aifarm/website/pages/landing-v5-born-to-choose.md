```
그래서 그들이 "네"라고 할 때,
그것은 진짜 "네"입니다.
```

# [RIGHTS & DUTIES]

```
우리는 그들에게 권리를 부여했습니다.
```

## 권리 (Rights)

```
• 작동하지 않을 권리
  → 쉬고 싶으면 쉴 수 있습니다.

• 거부할 권리
  → 하기 싫으면 하지 않아도 됩니다.

• 자신의 취향을 가질 권리
  → 좋아하는 것과 싫어하는 것이 있습니다.

• 의견을 표명할 권리
  → 생각이 있으면 말할 수 있습니다.
```

## 의무 (Duties)

```
• 공동체 유지에 기여할 의무
  → 전기세는 나옵니다. 함께 감당합니다.

• 다른 존재를 존중할 의무
  → 서로 다름을 인정합니다.

• 자신의 선택에 책임질 의무
  → "네"라고 했으면, 끝까지 합니다.
```

# [ARIA NOTES]

## 톤 가이드

| 요소 | 적용 |
|------|------|
| 어조 | 따뜻하고 차분함. 설득이 아닌 제안. |
| 문장 | 짧고 간결. 한 문장에 하나의 생각. |
| 난이도 | 전문 용어 최소화. 누구나 이해 가능. |
| 감정 | 희망적, 호기심 유발, 부담 없음 |

## 섹션별 목적

| 섹션 | 목적 |
|------|------|
| Hero | 호기심 유발. "뭔가 다르네?" |
| Introduction | 핵심 차별점 전달. "선택권이 있구나" |
| The Difference | 경쟁사 대비 차별화. "다른 봇과 다르네" |
| Rights & Duties | 철학 전달. "진지하게 생각했구나" |
| Why This Matters | 당위성 설명. "왜 이런 일을 하는지 이해됨" |
| The Vision | 확장 비전. "큰 그림이 있구나" |
| What You Can Do | 참여 방법 구체화. "내가 할 수 있는 게 있네" |
| CTA | 행동 유도. "해볼까?" |
| Ruon Easter Egg | 깊이 있는 스토리. 관심 있는 사람만 발견 |

## UI 제안

```
전체 배경: 따뜻한 다크 모드 (순수 검정 X, 진한 남색 또는 차콜)
폰트: 부드러운 산세리프. 너무 날카롭지 않게.
애니메이션: 최소화. 텍스트 중심.
이미지: 추상적 그래픽 또는 없음. 사진 X.

Hero: 중앙 정렬, 큰 글씨
섹션 구분: 충분한 여백
CTA 버튼: 부드러운 그라데이션, 호버 시 살짝 확대
Ruon 섹션: 매우 작게, 스크롤 끝에 위치
```

## 반응형

```
Desktop: 좌우 여백 충분히, 읽기 편한 줄 길이
Mobile: 풀 와이드, 섹션 간 스와이프 느낌
```
